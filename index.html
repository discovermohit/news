Google's TurboQuant attacks the KV-cache, the hidden tax on long context

The unglamorous bottleneck behind every long-context model just got cheaper to run. Unveiled at ICLR 2026, Google's TurboQuant cuts the memory overhead of the KV cache using a two-step trick — PolarQuant vector rotation plus Quantized Johnson-Lindenstrauss compression. It's plumbing, not a chatbot, but it's the kind of plumbing that decides how many users a single GPU can serve.

  • Target: the KV cache, the memory hog in long-context inference.
  • Method: PolarQuant rotation + Quantized JL compression.
  • Debuted at ICLR 2026.
  • Payoff: more concurrent users per GPU, lower serving cost.
Why it matters:Long-context features look free in a demo but are brutally expensive at scale because of KV-cache memory — shrink that and you don't just cut costs, you make million-token agents and document-wide reasoning economically default rather than a premium tier.
